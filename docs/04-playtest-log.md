# 플레이테스트 로그

실제 브라우저에서 게임을 플레이하며 확인한 기록입니다. 테스트는 로컬 HTTP 서버와 Headless Chromium 기반 Puppeteer 브라우저에서 진행했습니다.

## 2026-07-22 / 1차 기능 점검

환경: 데스크톱 viewport 1440×1000

- 페이지 제목과 게임 캔버스 로드 확인
- START RUN 클릭 후 `ready → playing` 전환 확인
- 첫 룰 카드 공개 확인
- ArrowRight 입력과 SPACE 대시 이벤트 전달 확인
- 약 8초 후 룰 카드가 첫 룰에서 다른 룰로 변경됨
- 점수·타이머·라운드 HUD 갱신 확인
- PAUSE 클릭 후 `playing → paused`, RESUME 클릭 후 `paused → playing` 확인
- 브라우저 콘솔 오류 없음
- 화면에서 플레이어·오브·드론·아레나 그리드·룰 HUD가 동시에 보이는지 시각 확인

## 2026-07-22 / 2차 전체 런 점검

환경: 데스크톱 viewport 1440×1000

- 실제 플레이 런을 종료 시점까지 진행
- 타이머가 `60.0`에서 `00.0`으로 감소
- 종료 시 `SHIFT COMPLETE` 오버레이 노출
- 종료 후 RUN IT BACK 클릭 시 새 런이 `ROUND 01`과 새 룰로 초기화

## 2026-07-22 / 반응형 점검

환경: 모바일 viewport 390×844

- `document.body.scrollWidth === window.innerWidth` 확인
- 가로 스크롤 및 레이아웃 넘침 없음
- 게임 카드·룰 스트립·시작 오버레이·조작 힌트가 화면 안에 표시됨

## 다음 테스트 항목

- GitHub Pages 배포 후 실제 공개 URL에서 재검증
- 30~60초 플레이 영상 녹화
- 규칙 카드별 체감 난이도와 점수 배율 조정

## 2026-07-22 / 프로덕션 개선 회귀 테스트

환경: 데스크톱 1440×1000, 모바일 390×844 및 320×700

- 전체 룰 수와 HUD 분모를 `08`로 통일
- 다음 룰 숫자 카운트다운이 8초부터 감소하는지 확인
- 드론을 주황색 화살촉 실루엣으로 변경해 오브와 구분되는지 시각 확인
- 대시 쿨다운 바·텍스트와 관성 방향 대시 동작 확인
- 오브 획득·충돌·대시 점수 팝업 생성 확인
- Web Audio 효과음 컨텍스트가 사용자 클릭 후 정상 실행되는지 확인
- 사운드 ON/OFF 설정과 최고 점수 localStorage 저장 확인
- 모바일 게임 카드를 첫 화면 상단으로 이동
- 모바일 캔버스가 1.6 비율을 유지하며 좌표 왜곡이 없는지 확인
- 모바일 방향키로 플레이어가 실제 이동하고 DASH가 작동하는지 확인
- 320px 및 390px에서 가로 스크롤 없음
- 키보드·터치 입력, 일시정지, 자동 일시정지, 종료·재시작 회귀 확인
