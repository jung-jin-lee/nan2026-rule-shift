# AI 활용 기술 문서

## 1. 활용 목적

RULESHIFT의 AI 활용은 게임의 재미와 완성도를 높이는 제작 보조에 초점을 둡니다. 런타임에 외부 LLM API를 호출하지 않으므로 심사자의 환경과 네트워크 상태에 관계없이 게임을 실행할 수 있습니다.

## 2. 활용 도구 및 내역

| 단계 | AI 활용 | 사람이 결정한 부분 |
| --- | --- | --- |
| 아이디어 | 60초 아케이드 핵심 루프와 룰 카드 후보 발산 | 최종 장르·목표·난이도 선택 |
| 기획 | 룰 카드의 문장, 카테고리, 안전장치 초안 작성 | 반복 방지·난이도·공정성 검수 |
| 개발 | HTML/CSS/JavaScript 구조와 캔버스 게임 루프 구현 보조 | 실제 코드 통합·동작 확인·수정 |
| UI | 방송 스튜디오 콘셉트, 카피, 화면 정보 우선순위 제안 | 색·레이아웃·타이포그래피 결정 |
| QA | 입력·타이머·일시정지·게임 종료 테스트 케이스 작성 | 브라우저에서 실제 플레이 및 버그 수정 |

## 3. 게임 안의 규칙 선택

게임 플레이 중에는 생성형 AI 또는 외부 API를 사용하지 않습니다. 룰 선택은 다음의 절차적 셔플 시스템으로 처리합니다.

1. 사전에 검수한 룰 카드 목록을 준비합니다.
2. 게임 시작 시 카드 순서를 섞습니다.
3. 최근 카테고리와 같은 카테고리는 후보에서 제외합니다.
4. 남은 후보 중 하나를 선택합니다.
5. 룰 변경 전 예고를 보여주고, 공개 후 플레이어에게 대응 시간을 제공합니다.

이 구조는 유저가 다음 룰의 정확한 패턴은 읽을 수 없게 하면서도, 불합리한 연속 난이도와 즉사 상황을 줄입니다.

## 4. 주요 프롬프트 기록

### 룰 카드 발산

```text
60초짜리 브라우저 생존 아케이드에 사용할 룰 카드를 제안해줘.
각 룰은 8초 동안 플레이어가 이해하고 대응할 수 있어야 하며,
이동·점수·적 행동·아레나 변화 중 하나만 바꿔야 한다.
같은 효과의 변형은 제외하고, 난이도와 공정성 위험도 함께 적어줘.
```

### QA 케이스

```text
Canvas 기반 웹 게임의 시작, 일시정지, 재시작, 키보드 입력,
60초 종료, 룰 변경, 충돌 점수 차감에 대한 수동 테스트 케이스를 작성해줘.
심사자가 설치 없이 브라우저에서 실행한다는 조건을 반영해줘.
```

## 5. 외부 에셋 및 라이선스

현재 버전은 외부 이미지·사운드 에셋을 사용하지 않습니다. 배경, 플레이어, 오브, 드론, 파티클은 Canvas API로 직접 그립니다. 외부 폰트는 Google Fonts의 Syne 및 DM Mono를 사용하며, 최종 제출 시 네트워크 없이도 보이게 할지 확인합니다.

## 6. 한계와 검수

AI가 제안한 룰은 그대로 사용하지 않고, 실제 플레이를 통해 다음 항목을 직접 확인합니다.

- 첫 플레이에서 목표와 조작이 이해되는가
- 룰 변경이 불공정하게 느껴지지 않는가
- 점수와 콤보가 즉시 피드백되는가
- 30~60초 영상에서 재미있는 변화가 드러나는가
